# JavaScript Execution Context

## Что такое контекст выполнения?

Контекст выполнения (Execution Context) — это абстрактное понятие в JavaScript, описывающее окружение, в котором выполняется код. Контекст выполнения содержит информацию о переменных, функциях, аргументах и области видимости, доступных во время выполнения кода.

## Типы контекстов выполнения

В JavaScript существует три типа контекстов выполнения:

1. **Глобальный контекст выполнения (Global Execution Context)** — создается при загрузке скрипта. Это базовый, самый внешний контекст, в котором выполняется код, не находящийся внутри функций.

2. **Контекст выполнения функции (Function Execution Context)** — создается каждый раз при вызове функции. Каждая функция имеет свой собственный контекст выполнения.

3. **Контекст выполнения eval (Eval Execution Context)** — создается при выполнении кода внутри функции `eval()`.

## Стек вызовов (Call Stack)

JavaScript использует структуру данных «стек» для управления контекстами выполнения. Это называется «стеком вызовов» (Call Stack). Принцип работы:

- Когда JavaScript начинает выполнение скрипта, он создает глобальный контекст и помещает его в стек вызовов.
- При вызове функции создается новый контекст выполнения и помещается на вершину стека.
- При завершении функции ее контекст удаляется из стека, и выполнение возвращается к предыдущему контексту.

```javascript
function first() {
  console.log('Inside first function');
  second();
  console.log('Again inside first function');
}

function second() {
  console.log('Inside second function');
}

first();
console.log('Inside Global Execution Context');
```

Порядок создания и удаления контекстов в стеке:
1. Добавляется глобальный контекст
2. Вызывается `first()` — добавляется контекст `first()`
3. Внутри `first()` вызывается `second()` — добавляется контекст `second()`
4. `second()` завершается — его контекст удаляется из стека
5. Продолжается выполнение `first()` — контекст остается в стеке
6. `first()` завершается — его контекст удаляется из стека
7. Продолжается выполнение глобального кода

## Фазы создания контекста выполнения

Создание контекста выполнения происходит в две фазы:

### 1. Фаза создания (Creation Phase)

На этой фазе происходит:

- Создание объекта переменных (Variable Object) / Лексического окружения (Lexical Environment)
- Создание цепочки областей видимости (Scope Chain)
- Определение значения `this`

#### Лексическое окружение (Lexical Environment)

Лексическое окружение — это внутренняя структура движка JavaScript, которая состоит из:

- **Environment Record (Запись окружения)** — хранит объявления переменных и функций
- **Ссылка на внешнее лексическое окружение** — ссылка на родительское окружение

```javascript
// Пример лексического окружения (концептуально)
lexicalEnvironment = {
  environmentRecord: {
    // Объявления переменных и функций
    // Включая аргументы для функционального контекста
  },
  outer: // ссылка на внешнее лексическое окружение
}
```

#### Hoisting (Поднятие)

Во время фазы создания происходит явление, называемое "поднятием" (hoisting). Объявления переменных и функций перемещаются в начало их области видимости:

- Объявления функций поднимаются вместе с их реализацией
- Переменные, объявленные через `var`, поднимаются со значением `undefined`
- Переменные, объявленные через `let` и `const`, также поднимаются, но находятся в «временной мертвой зоне» (Temporal Dead Zone) до момента фактического объявления.

```javascript
console.log(x); // undefined (из-за hoisting)
console.log(y); // ReferenceError: y is not defined
console.log(z); // ReferenceError: Cannot access 'z' before initialization

var x = 10;
let z = 30;

function example() {
  console.log("Это работает благодаря hoisting");
}
```

### 2. Фаза выполнения (Execution Phase)

На этой фазе выполняется код строка за строкой. Переменным присваиваются значения, и вызываются функции.

## Контекст this

Значение `this` определяется в момент вызова функции и зависит от того, как функция вызывается:

1. **Глобальный контекст**: В глобальном контексте `this` указывает на глобальный объект (в браузере это `window`, в Node.js — `global`).

```javascript
console.log(this); // window в браузере
```

2. **Контекст функции**: В обычной функции значение `this` зависит от способа вызова:

```javascript
function showThis() {
  console.log(this);
}

showThis(); // window в нестрогом режиме, undefined в строгом режиме

const obj = {
  method: showThis
};

obj.method(); // obj
```

3. **Стрелочные функции**: В стрелочных функциях `this` наследуется из окружающего лексического контекста:

```javascript
const obj = {
  regularMethod: function() {
    console.log(this); // obj
    
    const arrowFunc = () => {
      console.log(this); // obj (наследуется от regularMethod)
    };
    
    arrowFunc();
  }
};

obj.regularMethod();
```

4. **Явное указание контекста**: Можно явно указать значение `this` с помощью методов `call()`, `apply()` или `bind()`:

```javascript
function greet(greeting) {
  console.log(`${greeting}, я ${this.name}`);
}

const person = { name: 'Иван' };

greet.call(person, 'Привет'); // Привет, я Иван
greet.apply(person, ['Здравствуйте']); // Здравствуйте, я Иван

const boundGreet = greet.bind(person);
boundGreet('Добрый день'); // Добрый день, я Иван
```

## Замыкания (Closures)

Замыкание — это комбинация функции и лексического окружения, в котором эта функция была объявлена. Благодаря замыканиям, функция может иметь доступ к переменным из внешнего лексического окружения даже после завершения выполнения внешней функции.

```javascript
function createCounter() {
  let count = 0;
  
  return function() {
    return ++count;
  };
}

const counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3
```

В этом примере внутренняя функция сохраняет доступ к переменной `count`, даже когда контекст выполнения функции `createCounter` уже завершился и удален из стека вызовов.

## Лексическая область видимости и цепочка областей видимости

JavaScript использует лексическую область видимости, что означает, что область видимости определяется структурой кода во время объявления, а не выполнения.

Цепочка областей видимости (Scope Chain) — это список всех доступных лексических окружений, начиная с текущего и заканчивая глобальным. Когда движок JavaScript ищет переменную, он сначала проверяет текущее лексическое окружение, а затем последовательно внешние окружения в цепочке.

```javascript
const global = 'Я глобальная';

function outer() {
  const outerVar = 'Я из внешней функции';
  
  function inner() {
    const innerVar = 'Я из внутренней функции';
    console.log(innerVar); // Доступна из текущего окружения
    console.log(outerVar); // Доступна из внешнего окружения
    console.log(global);   // Доступна из глобального окружения
  }
  
  inner();
}

outer();
```

## Контекст выполнения в асинхронном JavaScript

Асинхронные операции в JavaScript (таймеры, промисы, async/await) добавляют сложности в управление контекстом выполнения. Для этого используется цикл событий (Event Loop) и очереди задач:

- **Цикл событий** контролирует выполнение кода, обработку событий и задач в очередях
- **Очередь задач (Task Queue)** содержит задачи, которые будут выполнены после завершения текущего стека вызовов
- **Очередь микрозадач (Microtask Queue)** имеет более высокий приоритет и обрабатывается перед очередью задач. Используется для промисов

```javascript
console.log('Start');

setTimeout(() => {
  console.log('Timeout');
}, 0);

Promise.resolve().then(() => {
  console.log('Promise');
});

console.log('End');

// Вывод:
// Start
// End
// Promise
// Timeout
```

## Оптимизация контекста выполнения

Понимание работы контекста выполнения позволяет оптимизировать код:

1. **Избегайте создания излишних замыканий** — они могут вызывать утечки памяти
2. **Используйте подходящий контекст this** — стрелочные функции для сохранения контекста, обычные функции для динамического контекста
3. **Учитывайте hoisting** — объявляйте переменные в начале области видимости
4. **Управляйте асинхронным кодом** — понимайте порядок выполнения асинхронных операций

## Заключение

Понимание контекста выполнения в JavaScript критически важно для написания эффективного и предсказуемого кода. Это фундаментальная концепция, которая объясняет многие особенности языка, включая замыкания, hoisting, лексическую область видимости и поведение `this`.

JavaScript имеет однопоточную природу, но асинхронные механизмы позволяют создавать неблокирующий код, функционирующий благодаря взаимодействию стека вызовов, цикла событий и очередей задач.
