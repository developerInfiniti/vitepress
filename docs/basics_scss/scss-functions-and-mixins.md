# SCSS Функции и Миксины (@function, @mixin)

SCSS предоставляет мощные возможности для определения собственных функций (`@function`) и миксинов (`@mixin`), что позволяет создавать переиспользуемый и параметризованный код стилей.

## Функции (`@function`)

Функции в SCSS позволяют определять пользовательские операции, которые принимают аргументы и возвращают значение. Они полезны для выполнения вычислений, преобразований или логических операций, результаты которых можно использовать в свойствах CSS.

**Синтаксис:**

```scss
@function function-name($arg1, $arg2, ...) {
  // Тело функции
  // ... вычисления ...
  @return $result;
}
```

* `@function`: Ключевое слово для определения функции.
* `function-name`: Имя вашей функции.
* `$arg1`, `$arg2`, ...: Аргументы, которые принимает функция (могут иметь значения по умолчанию).
* `@return`: Ключевое слово для возврата значения из функции.

**Пример:**

```scss
// Функция для преобразования пикселей в ремы
@function rem($pixel-value) {
  $base-font-size: 16px; // Базовый размер шрифта
  @return ($pixel-value / $base-font-size) + rem;
}

body {
  font-size: rem(16px); // Результат: 1rem
}

h1 {
  font-size: rem(32px); // Результат: 2rem
}

p {
  margin-bottom: rem(10px); // Результат: 0.625rem
}
```

**Функции с аргументами по умолчанию:**

```scss
@function calculate-padding($base, $multiplier: 1) {
  @return ($base * $multiplier) + px;
}

.element-default {
  padding: calculate-padding(10); // Использует значение по умолчанию для $multiplier (1) -> padding: 10px;
}

.element-large {
  padding: calculate-padding(10, 2); // Передает значение для $multiplier -> padding: 20px;
}
```

## Миксины (`@mixin`)

Миксины в SCSS позволяют определять группы объявлений CSS, которые можно повторно использовать (включать) в различные селекторы. Они могут принимать аргументы, что делает их очень гибкими для создания параметризованных стилей.

**Синтаксис:**

```scss
@mixin mixin-name($arg1, $arg2, ...) {
  // Группа объявлений CSS
  property: value;
  property: value;
  // ...
}
```

* `@mixin`: Ключевое слово для определения миксина.
* `mixin-name`: Имя вашего миксина.
* `$arg1`, `$arg2`, ...: Аргументы, которые принимает миксин (могут иметь значения по умолчанию).

**Включение миксина (`@include`):**

Чтобы использовать миксин в селекторе, применяется директива `@include`.

```scss
.selector {
  @include mixin-name($value1, $value2, ...);
  // Дополнительные стили для .selector
}
```

**Пример:**

```scss
// Миксин для задания border-radius с префиксами
@mixin border-radius($radius) {
  -webkit-border-radius: $radius;
     -moz-border-radius: $radius;
          border-radius: $radius;
}

.button {
  @include border-radius(5px);
  border: 1px solid black;
  padding: 10px 20px;
}

.alert {
  @include border-radius(10px);
  background-color: yellow;
  padding: 15px;
}
```

Компилируется в:

```css
.button {
  -webkit-border-radius: 5px;
     -moz-border-radius: 5px;
          border-radius: 5px;
  border: 1px solid black;
  padding: 10px 20px;
}

.alert {
  -webkit-border-radius: 10px;
     -moz-border-radius: 10px;
          border-radius: 10px;
  background-color: yellow;
  padding: 15px;
}
```

**Миксины с аргументами по умолчанию и необязательными аргументами:**

```scss
@mixin box-shadow($x: 0, $y: 0, $blur: 5px, $color: black) {
  -webkit-box-shadow: $x $y $blur $color;
     -moz-box-shadow: $x $y $blur $color;
          box-shadow: $x $y $blur $color;
}

.element1 {
  @include box-shadow(); // Использует значения по умолчанию -> box-shadow: 0 0 5px black;
}

.element2 {
  @include box-shadow(2px, 2px, 10px, rgba(0, 0, 0, 0.3)); // Передает все аргументы
}

.element3 {
  @include box-shadow($x: 5px, $blur: 15px, $color: red); // Передает аргументы по имени
}
```

**Включение содержимого блока в миксин (`@content`):**

Миксины могут принимать блоки стилей, которые затем можно вставить в миксин с помощью директивы `@content`. Это полезно для создания более абстрактных структур стилей.

```scss
@mixin responsive($breakpoint) {
  @media (min-width: $breakpoint) {
    @content; // Вставляет сюда стили, переданные при @include
  }
}

.container {
  width: 100%;

  @include responsive(768px) {
    width: 750px;
  }

  @include responsive(992px) {
    width: 970px;
  }

  @include responsive(1200px) {
    width: 1170px;
  }
}
```

## Разница между функциями и миксинами

* **Функции:** Предназначены для выполнения вычислений и возврата одного значения, которое можно использовать в качестве значения свойства CSS.
* **Миксины:** Предназначены для включения целых блоков объявлений CSS в селекторы. Они не возвращают значение напрямую, а генерируют стили.

Выбор между функцией и миксином зависит от того, нужно ли вам вычислить и вернуть значение или включить группу стилей.

## Когда использовать функции и миксины

* **Функции:**
    * Выполнение математических операций (например, преобразование единиц).
    * Генерация значений на основе логики (например, определение цвета на основе темы).
    * Получение значений из структур данных (например, из SCSS Maps).
* **Миксины:**
    * Группировка повторяющихся наборов свойств (например, префиксы для `border-radius`, `box-shadow`).
    * Создание параметризованных стилей (например, адаптивные медиа-запросы).
    * Включение сложных структур стилей.
    * Обработка вендорных префиксов.

## Заключение

Функции и миксины являются ключевыми инструментами в SCSS для создания DRY (Don't Repeat Yourself) и модульного кода стилей. Они позволяют абстрагировать повторяющуюся логику и наборы стилей, делая ваш CSS более организованным, легким в поддержке и масштабируемым. Правильное использование функций и миксинов значительно улучшает процесс разработки и качество вашего CSS.
